# Project Context — Cuban Food Restaurant Website

## Overview
A static website for a Cuban food restaurant, built with vanilla HTML, CSS, and JavaScript.
It is bundled with **Parcel v2** (zero-config bundler).
The project is an academic assignment (UOC — Front-end development, PEC1).

## Tech Stack
| Layer | Technology |
|-------|-----------|
| Markup | HTML5 |
| Styles | CSS3 (vanilla, no preprocessor by default) |
| Scripts | Vanilla JavaScript (ES modules) |
| Bundler | Parcel 2 (`parcel ^2.14.0`) |
| HTML partials | PostHTML + `posthtml-include ^2.0.1` |
| Package manager | npm |
| Node env | MacOS / zsh |

## Parcel Best Practices Applied to This Project

### Entry point
- The HTML entry is declared in `package.json` under the `"source"` field (`src/index.html`), which is the recommended Parcel v2 approach instead of passing it as a CLI argument.

### Asset referencing
- All assets (CSS, JS, images, fonts) **must be referenced relative to the HTML file** so Parcel can discover and process them automatically.
  - CSS: `<link rel="stylesheet" href="style.css">`
  - JS: `<script type="module" src="main.js"></script>`
  - Images: `<img src="./assets/images/photo.jpg">`
- Never use absolute paths like `/assets/...` for local files — they break Parcel's dependency graph.

### JavaScript modules
- Use **ES module syntax** (`import`/`export`) in `.js` files. Parcel handles transpilation and bundling automatically.
- Keep one `main.js` entry module imported from `index.html`; import everything else from there.

### CSS
- CSS files can be imported directly from HTML or from JS (`import './style.css'`).
- Parcel supports **CSS nesting**, **custom properties**, and **`@import`** out of the box.
- If a preprocessor is needed (Sass/Less), install the corresponding Parcel transformer (`@parcel/transformer-sass`) and rename the file — no extra config needed.

### Images and static assets
- Place images under `src/assets/images/`. Parcel will hash and copy them to `dist/` on build.
- For images that should NOT be processed (e.g. `favicon.ico`, `robots.txt`), place them in a `public/` folder and use the `--public-url` flag or the `staticFiles` plugin.

### Browserslist
- The `"browserslist"` field in `package.json` (`"> 0.5%, last 2 versions, not dead"`) controls transpilation targets for both JS (Babel) and CSS (Autoprefixer). Adjust as needed.

### Cache
- `.parcel-cache/` speeds up rebuilds. It is already in `.gitignore`. Delete it with `npm run clean` if you experience stale cache issues.

### Production build
- `npm run build` outputs optimised, minified assets with content hashes into `dist/`.
- Never deploy from `src/` — always deploy `dist/`.

### HTML Partials (`posthtml-include`)
- `posthtml-include` is installed as a dev dependency (`^2.0.1`). Parcel detects it automatically — no extra config file is needed.
- Use it to split repeated HTML fragments (header, nav, footer) into reusable partial files stored under `src/partials/`.
- Include a partial inside any HTML file with:
  ```html
  <include src="./partials/header.html"></include>
  ```
- The `src` path is **relative to the file containing the `<include>` tag**.
- Partials are plain HTML fragments — no `<!doctype>` or `<html>` wrapper needed.
- Changes to a partial trigger an automatic rebuild/HMR in dev mode.
- Data can be passed to a partial as attributes and accessed inside with `{{ attribute }}`.

### Multiple HTML pages
- To add a new page (e.g. `menu.html`), create `src/menu.html` and either:
  - Link to it from `index.html` (Parcel auto-discovers linked pages), or
  - Add it as a second entry in `package.json`: `"source": ["src/index.html", "src/menu.html"]`.

### Environment variables
- Use `.env` files (`.env`, `.env.local`) for environment-specific values.
- Access them in JS with `process.env.MY_VAR`. Parcel inlines them at build time.
- **Never commit `.env` files with secrets.** Add them to `.gitignore`.

## Domain Context — Cuban Food Restaurant

### Business concept
A restaurant website showcasing Cuban cuisine: traditional dishes, drinks (mojitos, Cuban coffee), and the cultural atmosphere of Cuba.

### Expected pages / sections
- **Home** (`index.html`) — Hero banner, tagline, highlights
- **Menu** — Food & drinks catalogue (categories: starters, mains, desserts, drinks)
- **About** — Story of the restaurant, chefs, Cuban culture
- **Gallery** — Photos of dishes and the venue
- **Contact / Reservations** — Address, phone, reservation form

### Tone & style
- Warm, vibrant, festive — reflecting Cuban culture
- Color palette suggestion: tropical tones (greens, yellows, reds) with earthy neutrals
- Typography: readable serif or display font for headings; clean sans-serif for body

### Language
- The website content is in **Spanish** (target audience is Spanish-speaking).
- Code, comments, and variable names should be in **English**.

## Coding Conventions
- **Indentation**: 2 spaces
- **HTML**: semantic elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<article>`)
- **CSS**: BEM-like class naming (`block__element--modifier`)
- **JS**: ES2020+, `const`/`let` (no `var`), arrow functions, async/await
- **No frameworks**: pure vanilla stack unless explicitly decided otherwise
- **Accessibility**: use ARIA roles, `alt` attributes on images, proper heading hierarchy

## What to Keep in Mind When Helping
- The project is at its very beginning — only the Parcel scaffold exists.
- Prefer zero-config Parcel solutions over adding extra tools.
- Keep the stack vanilla unless a concrete need arises.
- Respect the academic context: code should be clean, well-commented, and instructive.
- When adding files, always place source files under `src/` and follow the asset referencing rules above.
